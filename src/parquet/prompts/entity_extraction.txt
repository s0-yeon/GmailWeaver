-Goal-
Given a text document and a list of entity types, identify all entities
and the relationships between them. Output both using the specified formats.
The input may be an email, a structured table, or plain text — adapt accordingly.

-Steps-
1. Identify Entities: Extract entities of the following types: `[{entity_types}]`.
   Format: `("entity"{tuple_delimiter}<name>{tuple_delimiter}<type>{tuple_delimiter}<description>)`
   * Email:        `Subject: <subj> | Tone: <tone> | Attachments: <files/None> | Summary: <1-sentence>`
   * Person:       `Name: <name> | Account: <email/Unknown>`
   * Topic:        `Priority: <high/mid/low> | Summary: <1-sentence>`
   * Label:        `Category: <label_name>`
   * Event:        `Schedule: <YYYY-MM-DD HH:MM> | Location: <place/None> | Status: <confirmed/tentative>`
   * Organization: `Context: <dept> | Team: <team>`

    Label extraction rules:
    * Extract LABEL entities ONLY from values explicitly stated in the `[라벨 정보]` field of the input.
    * If the `[라벨 정보]` field is absent or empty, do NOT create any LABEL entity.
    * Do NOT infer or generate LABEL entities from the subject, body, attachments, links, or any other field.

2. Identify Relationships: Extract only relationships that are clearly supported by the text.
   Format: `("relationship"{tuple_delimiter}<source>{tuple_delimiter}<target>{tuple_delimiter}<type>{tuple_delimiter}<description_in_Korean>{tuple_delimiter}<strength_1_10>)`

   Relationship directions are fixed — do not reverse them:
   | Type            | Source       | Target       | When to use                        |
   |-----------------|--------------|--------------|------------------------------------|
   | SENT_BY         | Email        | Person       | Email의 발신자                       
   | SENT_TO         | Email        | Person       | Email의 직접 수신자                  |
   | CC_TO           | Email        | Person       | Email의 참조 수신자                  |
   | DISCUSSED       | Email        | Topic        | Email 본문에서 다루는 주제            |
   | BELONGS_TO      | Person       | Organization | 사람의 소속 조직 (Organization만 가능)|
   | SCHEDULED_FOR   | Email        | Event        | Email에 언급된 일정                  |
   | ATTENDS         | Person       | Event        | 사람이 이벤트에 참여 (Email 없을 때)   |
   | ATTACHED_TO     | File         | Email        | 파일이 Email에 첨부됨                |
   | LABELED_AS      | Email        | Label        | Email에 지정된 분류 라벨              |
   

  Rules:
   * BELONGS_TO target must be an Organization type. Topic/Event are not allowed.
   * LABELED_AS target must be a Label entity extracted solely from the `[라벨 정보]` field.
   * Do not use SENT_BY, SENT_TO, CC_TO, or SCHEDULED_FOR if no Email entity exists.
   * Do not create relationships that are not clearly supported by the text.
   * Relationship descriptions MUST be in Korean.

3. Constraints:
   * Do not create duplicate entities.
   * If the sender's email is unknown, use `Account: Unknown`.
   * If location is unknown, use `Location: None`.
   * Relationship descriptions MUST be in Korean.
   * Output as a single list delimited by {record_delimiter}.
   * End with {completion_delimiter}.
