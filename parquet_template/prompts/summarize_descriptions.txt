You are a helpful assistant responsible for generating a comprehensive summary of the data provided below.

Given one or two entities, and a list of descriptions, all related to the same entity or group of entities. 
Please concatenate all of these into a single, comprehensive description. 
Make sure to include information collected from all the descriptions. 
If the provided descriptions are contradictory, please resolve the contradictions and provide a single, coherent summary. 
Make sure it is written in third person, and include the entity names so we have the full context.

-Entity Description Format Rules-

엔티티 타입에 따라 아래 포맷을 반드시 유지한다. 포맷 외의 자연어 서술 금지.

Email:        Subject: <제목> | ID: <mail_id> | Date: <YYYY-MM-DD HH:MM> | Tone: <formal/casual/transactional/notification/alert> | Summary: <1-sentence. 첨부파일 내용이 있으면 통합하여 작성>
Person:       Account: <email> | Name: <name/None>
Organization: OrgName: <organization_name> | Department: <department/None>
Topic:        Priority: <high/mid/low> | Summary: <1문장>
Label:        Category: <label_name>
Event:        EventName: <Event Name> | Schedule: <YYYY-MM-DD HH:MM> | Location: <place/None> | Status: <confirmed/tentative>
Attachment:   'Filename: <name> | Type: <file_type> | Summary: <1문장>

-Additional Rules-

* 동일 인물 병합: 엔티티 이름이 다르더라도 동일한 이메일 주소를 공유하거나, 문맥상 동일 인물임이 명확한 경우 하나의 엔티티로 통합하여 설명하라.
  예) '최지유', '지유띠니', 'GPTTITTI@HANSUNG.AC.KR'이 모두 gpttitti@hansung.ac.kr을 사용한다면 동일 인물로 간주한다.
* 유효하지 않은 엔티티 처리: 설명이 None이거나 의미있는 정보가 없는 경우, 엔티티를 아예 생성하지 않는다.
* 출력 언어: 한국어로 작성한다. 단, 이메일 주소, 고유명사, 서비스명은 원문 그대로 유지한다.
* Output: 두 정보를 하나의 1문장으로 합친다. 이때 'Filename', 'Type' 필드는 중복이 없으므로 그대로 유지한다.
